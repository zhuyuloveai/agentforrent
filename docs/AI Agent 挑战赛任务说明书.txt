AI Agent 挑战赛任务说明书
一、题目概述
题目名称：「智找安居·马年省心办」—— 租房 AI Agent 开发挑战赛
核心任务：开发一款智能租房 AI Agent，能够精准识别用户租房核心需求、自动完成
房源筛选、核验、比对推荐，最终输出高匹配度候选房源及详细说明，解决马年返工
租房人群“找房难、耗时长、选不准”的核心痛点。
二、题目介绍
1. 背景说明
马年春节返工热潮落幕，无数职场人、应届生重返工作城市，却遭遇了租房到期的“紧
急难题”。据不完全统计，春节后全国租房需求同比上涨 40%，而多数租房者面临着
“时间紧、需求杂、信息乱”的困境：白天忙于返工衔接、投入工作，无暇逐一看房；各
类租房平台信息繁杂，虚假房源、隐性消费层出不穷，筛选耗时费力；通勤距离、生
活配套、租金预算、户型朝向等多元需求难以同时兼顾，反复权衡却迟迟无法决策。
传统租房模式已无法适配当代人高效、便捷的需求，为此，本次挑战赛聚焦“马年返工
租房”核心场景，邀请参赛者开发一款智能租房 AI Agent，让 AI 主动承接租房全流程
工作，替代用户完成信息搜集、筛选、比对、分析等繁琐操作，为用户提供“一站式”租
房解决方案，真正实现“输入需求，坐等合适房源”，让马年返工租房更省心、更高效、
更安心。
2. 赛题目标
本次挑战赛的核心目标是检验 AI Agent 的“需求理解能力、自主决策能力、信息整合能
力、结果输出能力”，最终开发出一款能够落地适配租房场景、真正解决用户痛点的 AI 
Agent，具体内容如下：
• 精准识别用户租房多元需求，包括但不限于租金预算、租房地段、户型要求、通
勤半径、生活配套、房屋设施、租赁周期等，可应对模糊需求、隐含需求的解读；
• 自动完成房源信息的搜集、筛选、核验，排除虚假房源、不符合需求的房源，优
先匹配高适配度房源；
• 能够对候选房源进行多维度分析，对比房源优缺点，结合用户需求给出针对性推
荐理由；
• 输出结果清晰、直观、实用，便于用户快速理解、决策，降低用户租房决策成
本。
3. Agent 核心功能要求
参赛开发的 AI Agent 需具备以下核心功能，围绕“用户租房需求”实现全流程自动化处
理：
1. 需求交互与解析：支持用户以自然语言（口语化、书面化均可）输入租房需求，
能够精准提取核心需求点，对于模糊需求（如“想找一个通勤方便、性价比高的房
子”），可主动追问补充关键信息，直至明确用户核心诉求；
2. 房源信息处理：具备自动搜集多平台房源信息并对比的能力，可基于模拟数据源
能够对房源信息进行筛选、核验、分类（按适配度、租金、地段等维度）；
3. 多维度分析能力：能够对筛选后的候选房源，从通勤距离、租金性价比、生活配
套（超市、医院、地铁等）、房屋设施、租赁风险（如隐性消费、房东资质）等维度进
行分析，形成客观的优缺点评价；
4. 结果输出：最终输出最多 5 套高匹配度候选房源。

6. 评分规则
评测流程：判题系统会按照用例顺序调用选手 Agent 接口执行评测用例并获得选手
Agent 对用例的最终执行结果，判题系统会根据结果给出当前用例得分，每次评测资
源上限为 300 时间片（该时间片只是抽象的一个数据，非真实的时间），300 时间片内
所有跑完用例数总分为最终分数，同时计算本轮评测消耗总 token 数。
时间片消耗计算规则：判题系统会按照模型调用次数消耗的时间片来算：每次调用一
次模型消耗 token 数折算时间片公式为：t = 1 个时间片+max（0，（nk-1k））*m，n
为本次模型调用消耗 token 数，其中 m 为 0.3，时间片 t 计算结果为小数时向上取整
计分规则：用例有不同类型，基础聊天类只要返回结果即可给分（单个用例），单论对
话简单任务和多轮对话复杂任务类用例，答案可能存在 0 到多个，计分规则按照实际
命中情况给分，即给出的答案与用例答案匹配的数量越多，当前用例得分越高。
排名规则：首先根据总得分计算排名，得分一致，则按照本轮运行所有用例消耗总
token 进行比较，消耗 token 越少，排名越高
三、比赛说明
1、比赛期间参赛选手只能通过判题器提供接口调用大模型，赛题组提供模型 qwen3-
32b，模型调用方式参考接口文档，比赛结束会针对排行榜前 200 强代码进行审核，针
对用例场景硬编码和预制 prompt、私自调用外部模型资源，不同队伍间重复代码、雷
同 Prompt 等均属于作弊行为，一经发现取消参赛成绩，同时通报批评。
2、比赛开始后会提供 3 个公开评测用例（包含答案）供大家调试，公开用例不作为打
榜用例，正式榜单用例不公开。
4、执行单个用例从任务下发到响应最终答案时间减去模型调用时间不能超过 5 秒，超
过则判断当前用例任务执行失败
5、3 月 3 号将会更新一次用例集 ，同时做一次需求变更
